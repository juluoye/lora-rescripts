# Branding And Attribution

This repository uses the project identity:

- Project name: `lora-rescripts`
- Official repository: `https://github.com/WhitecrowAurora/lora-rescripts`

## Allowed

- Referencing the project name to describe compatibility or lineage.
- Saying your fork is based on `lora-rescripts`.
- Linking to the official repository for attribution.

## Not Allowed Without Permission

- Presenting an unofficial fork, image, mirror, or hosted service as the official project.
- Suggesting endorsement, partnership, or approval by the original authors when none exists.
- Rebranding a modified build in a way that hides the original project origin from users.

## Recommended Fork / Mirror Labeling

If your build is modified, redistributed, mirrored, or hosted, label it clearly with wording such as:

- `Unofficial build based on lora-rescripts`
- `Modified fork of lora-rescripts`
- `Community mirror, not the official release`

## Why This Exists

The code is open under AGPL, but users should still be able to distinguish:

- the official project
- community forks
- mirrors and hosted variants
- modified builds with separate maintenance policies
